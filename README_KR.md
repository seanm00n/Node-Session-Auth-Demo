# 서버 사이드 세션 인증 시스템

> **"서버 측 상태 관리와 미들웨어 처리 파이프라인에 집중하여 구축한 Node.js 기반 인증 시스템입니다."**

[English Version / 영어 버전으로 읽기](./README.md)

---

## 📌 프로젝트 개요
본 프로젝트는 웹 환경에서의 **사용자 인증(Authentication)**과 **상태 유지(Session Management)**의 핵심 원리를 구현한 백엔드 시스템입니다. 서버 개발자로서 안전한 "요청 전처리 파이프라인"을 구축하고, MySQL 데이터베이스-서버 메모리-클라이언트 쿠키 간의 신뢰성 있는 데이터 동기화를 보장하는 데 집중했습니다.

## 🚀 주요 특징
* **미들웨어 파이프라인 아키텍처**: `express.json()` 및 `express-session`을 활용한 체계적인 요청 전처리 흐름 설계.
* **보안 세션 핸들링**: `HttpOnly` 및 `Secret Signing` 설정을 통해 XSS 공격 및 무단 변조를 방지하는 서버 측 세션 저장소 구축.
* **관계형 데이터베이스 연동**: MySQL을 활용한 실시간 사용자 정보 검증 및 세션 매핑.
* **인증 라이프사이클 구현**: 로그인, 세션 기반 접근 제어(대시보드), 로그아웃(세션 파괴 및 쿠키 회수) 기능 완비.

## 🛠 기술 스택
* **언어:** JavaScript (Node.js)
* **프레임워크:** Express.js
* **데이터베이스:** MySQL
* **라이브러리:** `express-session`, `dotenv`

## 🏗 시스템 아키텍처
`req` (요청) 객체를 동적인 작업 공간으로 접근했습니다. 모든 요청은 비즈니스 로직에 도달하기 전, 세션 데이터가 주입되는 "전처리 공정"을 거치도록 설계되었습니다.



## 🔧 트러블슈팅 및 배운 점
* **세션 초기화 순서**: 세션 미들웨어를 라우터보다 먼저 선언하여, `undefined` 속성 접근으로 인한 서버 크래시 문제를 해결하고 미들웨어의 순서 의존성을 파악했습니다.
* **비동기 데이터 처리**: `await response.json()`을 정확히 사용하여 프론트엔드와 백엔드 간의 비동기 통신 흐름을 최적화했습니다.
